NVIDIA Jetbot gestures recognition

Project Topic: Jetbot gestures recognition. The main idea is to make the robot classify several gestures. When a specific gesture is recognized, the robot responds with some action. It is easy to scale by adding new gestures and actions.

The link to the presentation can be found here.

The project consists of several parts. First, physical parts were assembled together and the robot was equipped with all the relevant software on an SD card. The 4 main functions of the robot are:

Diophantine Equations Solver

Solve Diophantine Equations (those where variables are integers) with constraints.

For this project I had to work intensively to create a working BNF grammar and a well-structured class-based parsing program.

By providing txt file with some structure, program parses this text to symbolic expressions. After the parsing the heavy-lifting part of actually solving the system is given to Z3 solver.

Here are some example of input-output of the program:

Where no solutions are found:

Semantic Similarity in Scientific Papers Using Transformers

In this project, we aim to use transformers to analyze the semantic similarity of scientific papers in the DIT publication database. By constructing a latent space for these papers, we can compute the Euclidean distances between their embeddings. This analysis will extend to finding the closest authors based on the semantic similarity of their research papers, potentially fostering new collaborations.

Objectives

1. Construct Latent Space: Use transformers to create embeddings for each paper abstract in the database.
2. Calculate Euclidean Distances: Determine the Euclidean distances between paper embeddings.
3. Author Pairing by Similarity: Find the distances between every possible pair of authors and recommend the top 5 closest authors for each author (excluding current co-authors).
4. Visualization and Clustering: Map the embeddings onto a 2D plane and apply k-means clustering.
5. Keyword Extraction and Wordclouds: Use KeyBERT to extract keywords from each cluster and display them as word clouds.

Steps

1. Data Preparation: Load and preprocess the JSON file containing paper abstracts and co-authors.
2. Constructing Latent Space: Generate embeddings for each abstract using a pre-trained transformer model.
3. Calculating Euclidean Distances: Compute the Euclidean distances between all pairs of paper embeddings.
4. Author Pairing: Aggregate paper embeddings for each author, compute distances, and identify the top 5 closest authors.
5. Visualization and Clustering: Reduce the dimensionality of embeddings, plot them on a 2D plane, and apply k-means clustering.
6. Keyword Extraction and Wordclouds: Extract keywords using KeyBERT and create word clouds for each cluster.

Tools and Libraries

Transformers (Hugging Face), SciPy, Scikit-Learn, KeyBERT, Matplotlib/Seaborn, WordCloud

Expected Outcome

This project will provide a detailed semantic analysis of scientific papers, facilitating new research collaborations by recommending authors based on the similarity of their work. Additionally, the visualization and keyword extraction will offer valuable insights into the major research themes within the database.

Simple RAG Project Using Board Games Dataset

This project involves creating a Retrieval-Augmented Generation (RAG) model using board games dataset. We utilized various datasets, including Monopoly, Battleship, Ticket to Ride, Codenames, and Exploding Kittens, to perform semantic analysis and generate responses based on the rules of these games.

Project Overview

In this project, we used transformers to embed the rules of several board games and performed clustering and keyword extraction to understand the semantic structure of the text. We also integrated a language model for generating responses based on user queries.

Data Import and Formatting

We imported data from various board games and performed text formatting to clean and prepare the text for analysis.

Embedding the Text

The cleaned text was split into sentences and then chunked into groups. We used the SentenceTransformer model to generate embeddings for these chunks.

UMAP Visualization

We used UMAP to reduce the dimensionality of the embeddings and create a 2D visualization of the sentence embeddings.

Clustering and Keyword Extraction

K-means clustering was performed on the reduced embeddings, and KeyBERT was used to extract keywords from each cluster. The keywords were visualized using word clouds.

Language Model Integration

We integrated the Gemma-7b-it language model from Hugging Face and Mistral-7b from GPT4ALL to generate responses based on user queries. The model was tested with various queries to ensure its accuracy and relevance.
